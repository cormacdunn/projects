def cleanstring(string):
    cleaned=''
    for i in string:
        if i not in ',()':
            cleaned += i
        else:
            cleaned = '0'
            break
    return cleaned

def listcleaner(listofstrings):
    cleanedlist = []
    for i in listofstrings:
        cleaned = cleanstring(i)
        if cleaned == '0.0':
            cleanedlist.append('0')
        if not cleaned:
            cleanedlist.append('0')
        else:
            cleanedlist.append(cleaned)
    return cleanedlist

# Create magnitude based on crime codes
# https://www.nap.edu/read/23492/chapter/7#129 -- resource used to classify crimes
# With the classifier function, we are checking whether the words in the crime code description have any match to our
# keywords, we then used positive matches to create a magnitude of severity. Values choosen with discretion. 

deathincluded = ['HOMICIDE']
actsofharm = ['ASSAULT', 'BATTERY', 'ABUSE', 'THREATS', 'KIDNAPPING', 'RESISTING', 'WEAPON']
injuroussexual =['SEXUAL', 'RAPE', 'PEEPING']
propertyonly = ['ARSON', 'CREDIT', 'FRAUD', 'COUNTERFEIT', 'EMBEZZLEMENT', 'SHOPLIFTING', 'THEFT', 'VANDALISM', 
                'STOLEN','BURGLARY']

def classifier(codestring):
    newstring = codestring.split()
    value = 0
    for i in newstring: 
        if i in deathincluded:
            value += 75
        if i in actsofharm:
            value += 20
            
        if i in injuroussexual:
            value += 30
        if i in propertyonly:
            value += 10
    return value/len(newstring)


latlong = latlong[latlong['Crime Code Description'].notnull()]
latlong['magnitude'] = latlong['Crime Code Description'].apply(classifier)

Given the Age and Crime Code, we can correctly predict the offenders's Gender 56.31% of the time with our training set and 56.03% with our test set. This suggest that there are more metrics to add to our model if we wish to more accurately estimate the gender of our culprit. Using Age and Crime Code is not enough to systematize our offender, and would be a dangerous and inaccurate way to proceed. With the EDA above, we did notice that some crimes and ages had a greater split in M/F proportion. By focusing on these metrics, or excluding metrics that have close to 50/50 split we would have a much more accurate model that may have practical uses. Because our dependent variable in this model is binary, we choose the Logistic Regression Model. With more time, it would be also be valuable to model with the Linear Probability Model and compare the results.¶
